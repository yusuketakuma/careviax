# `src/components/features/` ディレクトリ区分

`features/` 配下は機能（ドメイン）別のコンポーネント群。各ディレクトリを以下の2区分で分類する。

- **core（職種非依存）**: 薬局固有の業務ロジックに依存せず、他職種向け画面や汎用UIプリミティブとしても成立するコンポーネント群。多職種連携（医師・ケアマネ等）や管理系画面から再利用され得る。
- **pharmacy（薬局固有）**: 調剤・薬剤管理・処方箋など、保険薬局の業務ドメインに強く結びついたコンポーネント群。

区分はディレクトリ単位。ディレクトリ内に例外的に逆区分のコンポーネントが混在する場合は「主な区分」と「例外」を併記する。

## 分類の考え方

- **モデル依存性**: コンポーネントが薬局固有の Prisma モデル（`Intervention`, `ResidualMedication`, `MedicationIssue`, `DrugMaster` 等）や薬局業務プロセス（調剤・処方箋・医薬品マスタ）に直接結びついている場合は pharmacy。
- **props経由の汎用性**: 型・データを props 経由で受け取り、表示ロジック自体はドメイン非依存な場合（例: `cds/alert-panel` が `CdsAlert` を受け取るのみ）は core。
- **多職種連携前提の機能**: `docs/ph-os_pharmacy_multidisciplinary_collaboration_spec_project_context.md` に基づき、医師・ケアマネ等の他職種が閲覧・操作する機能（報告書、コメント、コラボレーション基盤）は core。
- **ディレクトリ内混在**: 大半のコンポーネントが core でも、ドメイン固有モデルに依存する少数のコンポーネントが混在する場合はディレクトリを core とし、例外として個別コンポーネントを注記する（新規追加時は将来的なディレクトリ分割を検討すること）。
