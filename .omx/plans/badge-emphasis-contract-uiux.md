# Badge / Emphasis Contract: UI/UX Unified Workflow Theme

## Purpose

Define a canonical contract for badge usage, severity emphasis, and count highlighting so the rollout stays calm, medically appropriate, and semantically consistent.

## Allowed Badge Uses

- workflow state
- backlog / stalled count
- approval / audit status
- urgency where action routing is actually different

## Forbidden Badge Uses

- decorative callouts with no state meaning
- repeating the same message that is already present in section headings
- stacking multiple competing badges for the same item without routing value

## Severity Levels

### Neutral

- Meaning: informational state, no immediate action
- Allowed style: outline / muted

### Attention

- Meaning: should be checked soon
- Allowed style: restrained accent, not alarm-heavy

### Urgent

- Meaning: blocked, overdue, or action-required now
- Allowed style: strongest allowed emphasis while still clinically calm

## Count Highlighting Rules

- show counts only when they change prioritization or routing
- do not repeat the same count in nearby surfaces without new meaning
- keep counts adjacent to the action link they are meant to trigger

## Copy Rules

- prefer operational wording over promotional wording
- keep labels short and scannable
- reuse the same state words across pages for the same meaning

## Review Rule

Any new badge or emphasis pattern introduced during the rollout must map to one of the categories above or be added here before broad adoption.
