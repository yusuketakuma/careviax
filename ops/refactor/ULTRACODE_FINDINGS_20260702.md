## F01 [24] FIX bug — Raw window query orders by created_at which the subquery does not expose - SQL error crashes the daily patient-status job

- File: `src/server/services/patient-status-tracker.ts:237` (finder: services-correctness)
- Evidence: The $queryRaw at lines 223-238 selects only `target_id, changes` in the inner query: `SELECT target_id, changes, ROW_NUMBER() OVER (PARTITION BY target_id ORDER BY created_at DESC) AS rn FROM "AuditLog" ...` then the outer query ends with `ORDER BY target_id, created_at DESC`. In PostgreSQL the outer ORDER BY can only reference output columns of the FROM subquery (target_id, changes, rn); created_at is not among them, so the statement fails with `column "created_at" does not exist`. The unit test mocks $queryRaw (`patient-status-tracker.test.ts:51 $queryRaw: vi.fn().mockResolvedValue(...)`) so the invalid SQL is never executed in tests. - Failure scenario: Daily job `patient_status_tracking` (src/server/jobs/daily/patient-status.ts calls trackPatientStatusChanges per org) runs against any org with at least one patient risk summary (patientIds non-empty, line 118 guard passes) -> $queryRaw throws PostgresError 42703 -> trackPatientStatusChanges rejects -> the job fails every run, so patient status-change detection and its notifications never fire in production.
- Suggested fix: Add created_at to the inner SELECT (matching patient-detail-documents.ts) or change the outer clause to `ORDER BY target_id, rn` (rn already encodes created_at DESC per target). Add a test that executes the SQL against a real Postgres (or at least snapshot-verifies the column list) instead of mocking $queryRaw.
- Verifier reason: Confirmed by reading patient-status-tracker.ts:223-238 (inner SELECT exposes only target_id, changes, rn; outer ORDER BY references created_at) and by executing the exact query shape against a real Postgres engine (PGlite 0.4.1 from repo node_modules): it fails with 'column "created_at" does not exist' (42703), while the sibling shape from patient-detail-documents.ts:597-608 (created_at included in inner SELECT) succeeds. patient-status-tracker.test.ts:51 mocks $queryRaw and its string assertions (lines 249-255) do not cover the outer ORDER BY, so no test asserts this as intended and the invalid SQL never runs in CI.
