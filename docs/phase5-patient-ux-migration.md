# Phase 5 患者 UI/UX 移行設計 (PRE-06)

## 概要

Phase 5 で患者モデルに追加・変更されるフィールド（構造化アレルギー・保険情報・gender enum・アーカイブ状態）を反映した患者詳細画面の情報アーキテクチャを定義する。
`docs/ui-ux-design-guidelines.md` の方針に準拠する。

---

## 情報アーキテクチャ設計

### 患者詳細画面の4ゾーン構成

```
┌─────────────────────────────────────────────────┐
│ ZONE 1: ページヘッダー（固定）                    │
│  氏名 / 氏名カナ / 生年月日・年齢 / 性別         │
│  [アーカイブ済みバナー]（該当時のみ）             │
├─────────────────────────────────────────────────┤
│ ZONE 2: サマリー帯（クリティカル情報）            │
│  ┌──────────┬──────────┬──────────┐            │
│  │ 重症アレルギー │ 最新検査値 │ 現在有効保険 │    │
│  └──────────┴──────────┴──────────┘            │
├─────────────────────────────────────────────────┤
│ ZONE 3: 主要作業エリア（タブ）                    │
│  [ケース] [薬剤] [スケジュール] [連絡先]          │
├─────────────────────────────────────────────────┤
│ ZONE 4: 補助情報（アコーディオン）                │
│  ▶ アレルギー詳細                                │
│  ▶ 保険情報詳細                                  │
│  ▶ 同意書                                        │
│  ▶ 基本情報（住所・電話）                        │
└─────────────────────────────────────────────────┘
```

---

## ZONE 1: ページヘッダー

### 表示項目と優先順位

| 優先度 | 項目 | データソース | 備考 |
|---|---|---|---|
| 1 | 氏名 | `patient.name` | 大フォント (20px) |
| 2 | 氏名カナ | `patient.name_kana` | サブテキスト (14px) |
| 3 | 生年月日・年齢 | `patient.birth_date` | 計算表示 (例: 1945-03-12 / 81歳) |
| 4 | 性別 | `patient.gender` (PatientGender enum) | アイコン+テキスト |
| 5 | ケース状態 | `careCase.status` | バッジ（色定義は後述） |

### 性別表示マッピング

| enum値 | 表示テキスト | アイコン |
|---|---|---|
| `male` | 男性 | ♂ |
| `female` | 女性 | ♀ |
| `other` | その他 | ⊕ |
| `unknown` | 不明 | ? |

### アーカイブ済みバナー（`patient.is_archived === true` 時のみ表示）

```
┌─────────────────────────────────────────────────┐
│ ⚠️  この患者はアーカイブ済みです               │
│  アーカイブ日: 2026-01-15　理由: サービス終了  │
│  [アーカイブを解除する]                         │
└─────────────────────────────────────────────────┘
```

- 背景色: `bg-amber-50` / ボーダー: `border-amber-300`
- アーカイブ患者では編集・新規ケース作成・スケジュール登録を無効化
- 解除操作には確認ダイアログを表示（破壊的操作防止）

## ZONE 2: サマリー帯（クリティカル情報）

3列グリッド。モバイルでは縦積み。

### カード 1: 重症アレルギー

```
┌──────────────────────┐
│ ⚠️ 重症アレルギー    │
│ ペニシリン系 (アナフィラキシー) │
│ ヨード造影剤 (重篤)   │
│ [詳細を見る →]       │
└──────────────────────┘
```

- データソース: `patient.allergies` (PatientAllergy[]) の `severity === 'severe'` または `'anaphylaxis'`
- 該当なし: 「重篤なアレルギーなし」を緑テキストで表示
- カード背景: アレルギーあり `bg-red-50` / なし `bg-green-50`
- severity 定義:

  | severity値 | 表示 | 色 |
  |---|---|---|
  | `anaphylaxis` | アナフィラキシー | 赤 (最高) |
  | `severe` | 重篤 | 赤 |
  | `moderate` | 中等度 | 橙 |
  | `mild` | 軽度 | 黄 |
  | `unknown` | 不明 | 灰 |

### カード 2: 最新検査値

```
┌──────────────────────┐
│ 📊 最新検査値        │
│ eGFR: 45.2 (2026-03) │
│ HbA1c: 7.8% (2026-02)│
│ [検査履歴 →]         │
└──────────────────────┘
```

- データソース: 最新の `PatientCondition` またはMCSサマリー
- 表示上限: 3件

### カード 3: 現在有効保険

```
┌──────────────────────┐
│ 🏥 保険情報          │
│ 医療保険: 123456789  │
│ 介護保険: 987654321  │
│ 有効期限: 2026-09-30 │
└──────────────────────┘
```

- データソース: `patient.insurances` (PatientInsurance[]) の `is_primary === true` かつ有効期限内
- 期限切れ保険: オレンジ警告「要更新」バッジ

| タブ | 内容 | Phase 5 変更 |
|---|---|---|
| ケース | CareCase 一覧・詳細 | アーカイブ患者は新規作成ボタン非表示 |

## ZONE 4: 補助情報（アコーディオン）

デフォルト閉じた状態。画面の情報密度を保つ。

### アレルギー詳細

Phase 5 前: `allergy_info: Json` を `<pre>` 表示（暫定）
Phase 5 後: 構造化テーブル表示

```
┌────────────────────────────────────────────────────┐
│ アレルゲン     | 反応            | 重症度 | メモ  │
│────────────────────────────────────────────────────│
│ ペニシリン系   | アナフィラキシー | 最重篤 |       │
│ ヨード造影剤   | 蕁麻疹・浮腫    | 重篤   |       │
└────────────────────────────────────────────────────┘
[アレルギーを追加]
```

### 保険情報詳細

```
┌──────────────────────────────────────────────────┐
│ 種別   | 番号       | 有効期限   | 状態          │
│────────────────────────────────────────────────── │
│ 医療保険 | 123456789 | 2026-09-30 | ✅ 有効       │
│ 介護保険 | 987654321 | 2026-03-31 | ⚠️ 要更新     │
└──────────────────────────────────────────────────┘
[保険情報を追加]
```

## アーカイブ患者の UI 状態設計

### 一覧画面での扱い

- デフォルト: アーカイブ患者を非表示
- フィルター: 「アーカイブ済みを含む」チェックボックスで表示切替
- 表示時: 行背景を `bg-gray-50`、氏名をグレーアウト、[アーカイブ] バッジ付与

### 詳細画面での制限

| 操作 | アーカイブ前 | アーカイブ後 |
|---|---|---|
| 患者情報閲覧 | ✅ | ✅（読み取り専用） |
| 患者情報編集 | ✅ | ❌（無効化） |
| 新規ケース作成 | ✅ | ❌（無効化） |
| ケース履歴閲覧 | ✅ | ✅ |
| スケジュール登録 | ✅ | ❌（無効化） |
| 連絡先編集 | ✅ | ❌（無効化） |
| PDF 出力 | ✅ | ✅（過去記録のみ） |
| アーカイブ解除 | — | ✅（権限者のみ） |

### アーカイブ操作のフロー

```
[アーカイブする] ボタン押下
  ↓
確認ダイアログ:
  「この患者をアーカイブしますか？
   アーカイブ後は編集・新規登録ができなくなります。
   [理由を入力（任意）] __________________
   [キャンセル] [アーカイブする]」
  ↓
POST /api/patients/[id]/archive
  ↓
一覧画面へリダイレクト + 成功トースト
```

## モバイル Wireframe 概要

### スマートフォン（375px）での表示

```
┌─────────────────────────┐
│ ← 患者一覧              │
│─────────────────────────│
│ 田中 花子               │
│ タナカ ハナコ            │
│ 1945-03-12 / 81歳 / 女性│
│ [稼働中]                │
│─────────────────────────│
│ ⚠️ ペニシリン系アレルギー│
│─────────────────────────│
│ 医療保険: 123456789     │
│─────────────────────────│
│ [ケース] [薬剤] [予定]  │  ← タブ（スクロール）
│─────────────────────────│
│  ケース一覧...          │
│                         │
│                         │
└─────────────────────────┘
```

- ZONE 2 サマリー帯: 重症アレルギーのみ常時表示、保険・検査値はアコーディオンに移動
- タブ: 水平スクロール（3タブ以上）
- 編集ボタン: 右上の `⋮` メニュー内（画面の広さ確保）
- タッチターゲット: 最小 44px × 44px

## 追加項目の表示優先順位（Phase 5 新規フィールド）

| 優先度 | 項目 | 表示場所 | 理由 |
|---|---|---|---|
| 1 | 重症アレルギー | ZONE 2 常時表示 | 薬剤安全に直結 |
| 2 | 有効保険情報 | ZONE 2 常時表示 | 請求・業務直結 |
| 3 | アーカイブ状態 | ZONE 1 バナー | 誤操作防止 |
| 4 | アレルギー詳細 | ZONE 4 アコーディオン | 詳細確認時のみ |
| 5 | 保険詳細・期限 | ZONE 4 アコーディオン | 詳細確認時のみ |
| 6 | 最新検査値 | ZONE 2（スペースあれば） | 参考情報 |
