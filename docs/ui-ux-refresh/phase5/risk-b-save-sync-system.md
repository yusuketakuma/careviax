## RB-01 自動保存（端末 Dexie）をサーバー保存と誤認

| 項目                    | 内容                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hazard                  | 端末内暗号化ドラフト（Dexie）への自動保存/「一時保存」を、サーバーに永続化された保存と誤認する                                                                                                                                                                                                                                                                                                                                     |

| Foreseeable use error   | 「一時保存」「下書き保存」の表示を見て他端末・他スタッフから参照可能と誤認し、送信せず放置・端末返却する                                                                                                                                                                                                                                                                                                                           |

| Potential harm          | 訪問記録・薬歴の喪失 → 次回訪問時の残薬/副作用/観察情報の欠落、報告書・算定根拠の欠落、再訪問                                                                                                                                                                                                                                                                                                                                      |
| Severity                | 中（記録欠落経由の間接的な患者影響）                                                                                                                                                                                                                                                                                                                                                                                               |
| Probability             | 高（訪問業務の日常操作。オフライン前提の運用）                                                                                                                                                                                                                                                                                                                                                                                     |
| Detectability           | 中（未同期件数はヘッダーに出るが語彙自体は無警告）                                                                                                                                                                                                                                                                                                                                                                                 |
| Existing control        | 端末保存の明示文言の先例: `capture-content.tsx:250`・`voice-memo-content.tsx:214`（「端末に保存しました（通信がなくても残ります）」）。鍵欠如 fail-close 文言: `visit-record-form.tsx:899`、`prescription-intake-form.tsx:578`。オフライン手動保存 toast「オフラインで下書きを保存しました。再接続後に自動同期します」`visit-record-form.tsx:1836`。自動保存フック `src/lib/hooks/use-soap-draft.ts` / `use-prescription-draft.ts` |

| Verification method     | 保存バー文言・状態の unit test。実ネットワーク断での E2E（J-10 は現状 route-mocked のみ — 実断 E2E は未実装、`tools/tests/ui-route-mocked-smoke.spec.ts:2215`）                                                                                                                                                                                                                                                                    |

| Review owner            | design（主）・clinical safety                                                                                                                                                                                                                                                                                                                                                                                                      |

| 項目                    | 内容                                                                                                                                                                                                                                                       |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| Cause                   | 「同期済み HH:MM」は `useOfflineStore.lastSyncedAt` = **最後に drain が走った時刻**であり、キューが空であることの保証ではない（`src/components/layout/app-header.tsx:67-98`）。同期状態の情報が HeaderSyncStatus と OfflineDraftIndicator（同 :114）に分裂 |
| Potential harm          | 未同期の訪問記録・観察データが端末に滞留 → RB-01/RB-04 経由で喪失、他スタッフが記録なしと誤解して重複対応                                                                                                                                                  |
| Severity                | 中                                                                                                                                                                                                                                                         |
| Probability             | 中                                                                                                                                                                                                                                                         |
| Detectability           | 中（`OfflineDraftIndicator` の件数表示 `src/components/features/offline/offline-draft-indicator.tsx` はあるが、緑表示と並存して打ち消し合う）                                                                                                              |
| Existing control        | 未同期件数インジケータ（app-header.tsx:114）、`/offline-sync` の同期キュー一覧・競合ビュー（`offline-sync-content.tsx`）、online イベントでの自動 drain（`sync-engine.ts`）、notifications の未同期合成行（inv-10）                                        |
| Required UI control     | ヘッダーの同期表示を**単一の状態機械**に再設計: 「未同期 n 件（橙・件数リンク→/offline-sync）/ 競合 n 件（赤）/ すべて同期済み（緑）/ オフライン（赤）」。キュー非空のときに「同期済み」の緑を出さない。lastSyncedAt は補助情報に降格                      |
| Required system control | 24h 超の滞留ドラフト・未解決競合をログイン時モーダル/通知でエスカレーション（サーバーは端末内容を知り得ないため端末側で実装）                                                                                                                              |
| Verification method     | HeaderSyncStatus の状態遷移 unit test（pending>0 で緑を出さない）、競合シード（`__phosSeedOfflineSyncDemo('conflict')`）での表示 E2E                                                                                                                       |
| Residual risk           | ユーザーがバッジ自体を見ない可能性（配置・コントラストの design review 対象）                                                                                                                                                                              |
| Review owner            | design（主）・clinical safety                                                                                                                                                                                                                              |

| Detectability           | 低（`writePatientFieldRevisions`（同 route.ts 内）で履歴は監査可能だが「静かに」起きる — 事後追跡は可、事前検知ゼロ）                                                                                                                                                                                                                                                                                                                                                                                                         |
| Existing control        | コア臨床フローは OCC 済: visit-records `expected_version`（`src/app/api/visit-records/route.ts:1085`）、dispense/set 系 cycle.version（`dispense-results/route.ts:542`、`dispense-audits/route.ts:658`）、handoff（`visit-records/[id]/handoff/route.ts:136`）。オフライン競合は 409 → `server_conflict` 退避（`sync-engine.ts:378`）→ ConflictDiffDialog + use_server/use_local 明示解決 + ローカル上書きガード（`offline-sync-content.tsx:293,368,376-381`）— **2026-07-09 レビュー済み着地。本登録簿はこの設計を覆さない** |
