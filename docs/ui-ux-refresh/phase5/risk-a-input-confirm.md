# Phase 5 Use-error Risk Register — A: 入力・確定・識別系

作成日: 2026-07-11 / 作成方法: read-only コード検証（実画面ブラウザ確認なし。file:line は本日時点の実測）。
入力ソース: `04-screen-and-state-inventory.md` / `05-state-ownership.md` / `phase3/user-journeys.md` / `01-scope-and-users.md` + 実コード突合。

判定基準:

- **Severity（患者影響）**: 高=誤投薬・禁忌見落とし等の直接的健康被害につながり得る / 中=治療遅延・記録齟齬など間接影響 / 低=業務混乱に留まる。
- **Probability**: 実装・運用フローから見た発生しやすさ（高・中・低）。
- **Detectability**: エラーが患者到達前に発見される見込み（高=後工程で構造的に捕捉 / 低=捕捉機構なし）。
- **Residual risk** は「Required control 実装後」ではなく**現状**の残余リスク。
- 既知の意図的仕様（ConfirmDialog autoFocusConfirm、org-wide アクセス、React Compiler 等）は問題として扱わない。
