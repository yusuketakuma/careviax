## 1. 電子処方箋管理サービス

- **正式名称**: 電子処方箋管理サービス（電子処方箋の仕組み。オンライン資格確認等システムを基盤とし、医療介護総合確保法に基づき運用）
- **発行主体**: 厚生労働省（制度所管）/ 実施機関: 社会保険診療報酬支払基金・国民健康保険中央会（国保連）
- **バージョン/公表日（確認できた版）**:
  - システムベンダ向け技術解説書 **2.04版**（令和8年7月、2026-07-03 掲載）
  - 薬局向け電子処方箋導入スターターキット **第2.0版**（2026-06-15 掲載）
  - リフィル処方箋対応: 2023-12-28 機能開始 / 院内処方対応: 2025-01-23 機能開始
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/stf/denshishohousen.html （厚労省 電子処方箋ポータル）
  - https://www.mhlw.go.jp/content/11121000/001569995.pdf （第5回電子処方箋推進会議 資料1、令和7年9月29日）
  - https://www.digital.go.jp/en/resources/govdashboard/electronic-prescription （デジタル庁 導入状況ダッシュボード）
- **対象範囲**: 全保険薬局。処方箋の電子受付、処方・調剤情報の参照、重複投薬等チェック、調剤結果登録、リフィル処方箋管理。
- **適用性**: **Mandatory（事実上）**。法律上の一律応需義務規定は確認できていないが、2026年時点で薬局導入率は9割超（厚労省推進会議資料・業界報道）であり、調剤報酬の施設基準（後述 §9）で電子処方箋応需体制＋調剤結果の原則全件登録が要求されるため、SaaS としては必須前提と扱う。
- **適用性判断の根拠**: 厚労省推進会議資料（2025-09-29）が「今夏時点で概ねすべての薬局に導入見込み」と明記。2026年度調剤報酬改定の体制加算施設基準に調剤結果登録が組み込まれた。
- **UIへの含意**:
  - 処方受付 UI は「電子処方箋（引換番号＋マイナ/資格確認起点）」と「紙処方箋」の**二系統受付**を常設し、紙受付でも調剤結果登録が必要な運用（原則全件登録）に対応する。
  - 調剤完了フローの終端に「調剤結果登録（速やかに）」ステップを組み込み、未登録件数をワークリスト/バッジで可視化する。
  - 重複投薬等チェック結果（管理サービス側チェック）の表示領域と、薬局内 CDS（PH-OS 自前チェック）との**出所区別表示**が必要。
  - リフィル処方箋は残回数・次回調剤予定日のライフサイクル管理 UI が必要。
  - 調剤済み電子処方箋は支払基金/国保連の「調剤済み処方箋の保存サービス」（5年間保存）を利用可能 → 薬局内保存と外部保存のステータス表示。
  - 令和8年度に管理サービス内医薬品マスタ（YJ/レセ電/一般名コード対応関係）の公開が予定されており、PH-OS の医薬品マスタ結合設計（SSK/HOT）との突合機会になる。

## 2. オンライン資格確認

- **正式名称**: オンライン資格確認（オンライン資格確認等システム）。根拠: 保険医療機関及び保険医療養担当規則・保険薬局及び保険薬剤師療養担当規則等の改正（令和4年改正、原則義務化）
- **発行主体**: 厚生労働省保険局 / 実施機関: 社会保険診療報酬支払基金・国保中央会
- **バージョン/改定日（確認できた内容)**: 2023-04-01 原則義務化施行。2025-09-19 よりスマートフォン搭載マイナ保険証の読み取り機能が全医療機関等へ開放。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/stf/newpage_08280.html （厚労省 オンライン資格確認・システムベンダ向け）
  - https://www.mhlw.go.jp/stf/newpage_50657.html （厚労省 資格確認方法について）
  - https://www.med.or.jp/nichiionline/article/010835.html （日医オンライン: 療担規則改正の解説。二次資料だが改正内容の要点確認用）
- **対象範囲**: 全保険薬局（紙レセプト請求が認められる例外機関を除く）。患者がマイナ保険証による資格確認を求めた場合の対応体制整備を含む。
- **適用性**: **Mandatory**。保険薬局である限り原則義務。
- **適用性判断の根拠**: 療担規則・薬担規則の改正により 2023-04-01 から導入が原則義務化され、患者がオンライン資格確認を求めた場合は当該方法で確認しなければならない。
- **UIへの含意**:
  - 受付 UI に資格確認結果（有効/無効/要確認、保険者情報、負担割合）の表示と、資格エラー時の差戻し導線が必要。
  - **在宅訪問（PH-OS の主戦場）では「居宅同意取得型」オンライン資格確認**（訪問診療・訪問薬剤管理指導等向け、モバイル端末で同意取得・資格確認する方式。厚労省 newpage_08280 掲載）への対応が競争上重要。訪問先でのモバイル資格確認フローと同意取得記録の保存 UI を設計対象にすべき。
  - スマホ搭載マイナ保険証（2025-09〜）により、対面受付の読み取り手段が多様化。受付手段の選択 UI は「カード/スマホ/従来保険証（経過措置）/資格確認書」を想定する。

## 3. 調剤済み処方箋の保存（薬剤師法第27条）

- **正式名称**: 薬剤師法（昭和35年法律第146号）第27条（処方箋の保存）
- **発行主体**: 国（厚生労働省所管）
- **バージョン/改定日**: 現行: 調剤済みとなった日から**3年間**保存（昭和35年制定以来）。**改正**: 「医薬品、医療機器等の品質、有効性及び安全性の確保等に関する法律等の一部を改正する法律」（**令和7年法律第37号**、2025-05-21 公布）により**5年間**へ延長。施行期日は**公布後2年以内の政令で定める日**（2026-07-11 時点で施行日告示は確認できず＝**未施行の可能性が高い。要再確認**）。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/stf/newpage_58083.html （厚労省 令和7年薬機法等改正ページ）
  - https://www.mhlw.go.jp/content/11121000/001311524.pdf （厚科審 医薬品医療機器制度部会 資料4、令和6年10月3日 — 3年保存の現状と5年延長の論点）
  - https://www.mhlw.go.jp/content/11121000/001277224.pdf （薬局に係るその他の課題について、令和6年7月19日）
- **対象範囲**: 全薬局。紙・電子を問わず調剤済み処方箋。
- **適用性**: **Mandatory**（+ 5年延長部分は Requires legal review — 施行日確定の追跡が必要）。
- **適用性判断の根拠**: 薬剤師法の直接の保存義務。保険薬局には薬担規則第6条（処方箋・調剤録を完結の日から3年間保存）も重畳適用。
- **UIへの含意**:
  - 記録ライフサイクル: 保持期限の起算は「調剤済みとなった日」。レコードごとに**起算日と保持期限を計算・保持**し、期限は法改正で3年→5年に変わるため**保持ポリシーを設定値（テナント共通のシステム設定）として外出し**する。施行日をまたぐレコードの扱い（施行日時点で保存中のものが5年になるか等は経過措置次第）は施行時に法務確認。
  - 削除/廃棄は保持期限前は不可（UI 上ブロック）、期限後も監査ログを残す運用に。
  - 電子処方箋は管理サービス側の保存サービス（5年）との二重管理状態を UI で明示できるとよい。

## 4. 調剤録の保存（薬剤師法第28条・薬担規則第6条）

- **正式名称**: 薬剤師法第28条（調剤録: 記入・保存義務）/ 保険薬局及び保険薬剤師療養担当規則第6条（処方箋・調剤録の保存）
- **発行主体**: 国（厚生労働省所管）
- **バージョン/改定日**: 現行: 最終の記入の日から**3年間**保存。令和7年法律第37号により**5年間**へ延長（施行期日は上記 §3 と同じく公布後2年以内・2026-07-11 時点で施行日未確認）。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/content/11121000/001311524.pdf （制度部会資料: 薬剤師法27条・28条の3年保存は制定以来未改正、5年への延長方針）
  - https://www.mhlw.go.jp/stf/newpage_58083.html （令和7年薬機法等改正）
- **対象範囲**: 全薬局。保険調剤では調剤報酬点数等の記載事項が加わる（薬剤師法施行規則・薬担規則）。
- **適用性**: **Mandatory**。
- **適用性判断の根拠**: 薬剤師法の直接義務。電子調剤録は e-文書法（§6）により許容され、厚労省解説（令和6年11月）は「レセコン＋電子薬歴」で調剤録の保存要件を満たす構成を明示的に許容。
- **UIへの含意**:
  - PH-OS の調剤記録（dispense/audit 工程の記録）は**法定調剤録の記載事項を充足するデータモデル**であることを明示し、「調剤録として出力（見読性確保）」できる印刷/表示ビューが必要（§6 の見読性要件）。
  - 追記・訂正は**訂正履歴を保持した上での追記型**（真正性: 改変・消去の事実と内容を確認できる措置、責任の所在の明確化）。UI は「上書き」ではなく「訂正版の追加＋訂正理由」を促す設計に。
  - 保持期限計算の起算は「最終の記入の日」— 処方箋（調剤済み日起算）と**起算日が異なる**点をライフサイクル管理で区別する。

## 5. 薬剤服用歴・服薬指導記録の保存（薬機法施行規則・体制省令）

- **正式名称**:
  - 医薬品医療機器等法施行規則（昭和36年厚生省令第1号）第15条の14の3（調剤した薬剤に関する情報提供・指導の記録: 「情報の提供及び指導の内容の要点」等）
  - 薬局並びに店舗販売業及び配置販売業の業務を行う体制を定める省令（体制省令）: 服薬指導等の記録を**最終記入日から3年間**保存（令和2年（2020年）9月施行の改正で義務付け）
  - 調剤報酬上の薬剤服用歴（服薬管理指導料等の算定要件）: 最終記入日から3年間保存
- **発行主体**: 厚生労働省
- **バージョン/改定日（確認できた内容）**: 服薬指導記録の3年保存義務付けは 2020年改正（改正薬機法の施行に伴う省令改正）。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://laws.e-gov.go.jp/law/336M50000100001 （e-Gov: 薬機法施行規則）
  - https://www.mhlw.go.jp/content/11121000/001277224.pdf （厚労省: 薬局の記録・保存義務の整理資料）
  - https://www.cbnews.jp/news/entry/20200609181842 （二次資料: 2020年省令改正で服薬指導記録3年保存義務付けの報道。条文確認は e-Gov で実施）
- **対象範囲**: 全薬局の薬剤服用歴・服薬指導記録。
- **適用性**: **Mandatory**。付随論点は Requires legal review:
  - 自立支援医療・生活保護・難病等の公費関連薬歴は**5年保存**が求められる（各公費制度側の規定。確認できたのは二次資料 https://www.phchd.com/jp/medicom/park/idea/ph-management-keep-medicationhistory であり、制度別の一次確認が必要）。
  - 調剤報酬請求権の消滅時効5年（2020年民法改正）に合わせ薬歴も5年保存が望ましいとする実務見解あり（義務ではない）。
- **適用性判断の根拠**: 薬機法施行規則・体制省令の直接義務＋調剤報酬算定要件。
- **UIへの含意**:
  - 薬歴（PH-OS の服薬指導・薬歴モジュール）は**記録種別ごとに保持期限が異なりうる**（通常3年/公費関連5年/実務推奨5年）ため、患者の公費区分に連動した保持分類をレコードに付与できる設計が安全。実装は「一律5年以上保持」に倒すのが最も単純で、法的下限を全て満たす。
  - 「最終記入日」起算のため、追記があるたびに保持期限が延伸される — 記録クローズ（完結）概念と最終記入日の管理が必要。
  - 服薬指導記録の必須記載事項（指導内容の要点、患者の状況等）を入力フォームの必須/推奨フィールドとして構造化する。

## 6. e-文書法＋e-文書法厚生労働省令（電磁的記録による保存の要件）

- **正式名称**:
  - 民間事業者等が行う書面の保存等における情報通信の技術の利用に関する法律（平成16年法律第149号、e-文書法）
  - 厚生労働省の所管する法令の規定に基づく民間事業者等が行う書面の保存等における情報通信の技術の利用に関する省令（平成17年厚生労働省令第44号）
  - 解説資料: 「薬局内における文書の電子化等について（解説）」厚生労働省医薬局総務課、**令和6年11月**
- **発行主体**: 国 / 厚生労働省
- **バージョン/公表日（確認できた版）**: 解説資料は令和6年11月版。省令44号は e-Gov 現行版（https://laws.e-gov.go.jp/document?lawid=417M60000100044）。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/content/001279081.pdf （薬局内における文書の電子化等について（解説）— 本調査で全文確認）
  - https://laws.e-gov.go.jp/document?lawid=417M60000100044 （e-Gov: 厚労省令44号）
- **対象範囲**: 処方箋・調剤録をはじめ、麻薬帳簿・覚醒剤帳簿等、厚労省所管法令上の保存義務文書の電磁的保存/作成/交付。薬剤情報提供書（薬情）の電子交付は薬機法第9条の4第1項で許容。外部保存（クラウド）も3要件充足を条件に可。
- **適用性**: **Mandatory**（PH-OS は電子保存が前提のため、電子保存を適法にする根拠であり、その要件は必須）。
- **適用性判断の根拠**: 厚労省解説が「処方箋・調剤録は e-文書法＋省令44号に基づき電子的に取扱い可能」「電子保存には3要件（見読性・真正性・保存性）＋運用管理規程＋個情法配慮＋サイバーセキュリティ対策が必要」「医療情報システムの安全管理に関するガイドラインを参照」と明記。
- **UIへの含意（記録ライフサイクルの中核要件）**:
  - **見読性**: 保存期間中いつでも「直ちに明瞭かつ整然とした形式で」画面表示・書面出力できること（省令44号第4条第4項第1号）。→ 全法定記録に印刷可能な帳票ビュー必須。アーカイブ層に移しても即時表示可能に。
  - **真正性**: 「改変又は消去の事実の有無及びその内容を確認できる措置」＋「作成責任の所在の明確化」（同第2号）。→ 追記型監査証跡（誰が・いつ・何を）、記録作成者/確定者の表示、確定（サイン/finalize）操作の UI。電子作成で署名が必要な文書は電子署名法上の電子署名（省令第7条）。
  - **保存性**: 保存期間中「復元可能な状態」で保存（同第3号）。→ バックアップ/フォーマット継続性はインフラ要件だが、UI 的には保持期限とアーカイブ状態の可視化。
  - 麻薬帳簿等（麻薬及び向精神薬取締法第37〜40条の帳簿）も省令44号の別表対象であり電子化可能 — 在庫/帳簿系モジュールにも同じ3要件パターンを適用。
  - 医療情報システムの安全管理に関するガイドライン（v6.0、PH-OS 既対応）が技術・運用面の SSOT である点は既存方針と整合。

## 7. 電子帳簿保存法

- **正式名称**: 電子計算機を使用して作成する国税関係帳簿書類の保存方法等の特例に関する法律（平成10年法律第25号、電子帳簿保存法）
- **発行主体**: 国（国税庁所管）
- **バージョン/改定日**: 令和3年度改正で電子取引データの電子保存が義務化、宥恕措置終了により **2024-01-01 から完全義務化**。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.nta.go.jp/law/joho-zeikaishaku/sonota/jirei/index.htm （国税庁 電子帳簿保存法関係）
  - https://laws.e-gov.go.jp/law/410AC0000000025/ （e-Gov: 法律本文）
- **対象範囲**: **国税関係帳簿書類および電子取引の取引情報**（請求書・領収書・見積書・契約書等）。調剤録・薬歴・処方箋などの**医療記録そのものは国税関係書類ではなく本法の対象外**（本法の対象定義から判断。医療記録の電子保存根拠は §6 の e-文書法系）。
- **適用性**: **Conditionally applicable**。PH-OS が医薬品仕入・経費関連の電子取引データ（電子請求書等）を保管する機能（例: 仕入管理・発注）を持つ場合のみ、検索要件（取引年月日・金額・取引先での検索）・改ざん防止措置等が適用される。現行スコープ（調剤・薬歴・訪問報告）には非適用。
- **適用性判断の根拠**: 本法の対象は国税関係帳簿書類に限定される。薬局経営側の経理文書は対象（税理士系解説多数）だが、それは薬局の経理システムの領域。
- **UIへの含意**: PH-OS に発注・仕入エビデンス保管機能を追加する場合は、電子取引データの検索3要件（日付・金額・相手先）を満たす検索 UI とタイムスタンプ/訂正削除履歴等の要件検討が必要になる。それまでは設計上の将来留意点に留める。

## 8. レセプト（調剤報酬）オンライン請求

- **正式名称**: 療養の給付及び公費負担医療に関する費用の請求に関する命令（昭和51年厚生省令・内閣府令、通称「請求命令」）— 令和5年改正によりオンライン請求を原則化
- **発行主体**: 国（厚生労働省）/ 審査支払機関: 社会保険診療報酬支払基金・国保連
- **バージョン/改定日（確認できた内容）**:
  - オンライン請求原則化: 光ディスク等請求は令和6年（2024年）9月30日まで（以降は移行計画書届出が必要）。書面請求は限定的例外（レセコン未使用または常勤薬剤師が昭和19年4月1日以前生まれ等、令和6年2月29日までの届出）。
  - 移行目標: オンライン資格確認導入済み機関は**2026年（令和8年）9月末までにオンライン請求へ移行**（社会保障審議会了承のロードマップ、2023-03-23）。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.ssk.or.jp/seikyushiharai/iryokikan/rezept_03.html （支払基金: オンライン請求の原則について — 本調査で内容確認）
  - https://www.ssk.or.jp/seikyushiharai/iryokikan/rezept_02.html （支払基金: 請求命令について）
  - https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/0000190624_00001.html （厚労省: オンライン請求等）
- **対象範囲**: 保険薬局の調剤報酬請求（レセプト電算処理システム、CSV 形式の調剤レセプト）。
- **適用性**: **Mandatory**（PH-OS 自体がレセプト請求送信を担うか、レセコン連携かはアーキテクチャ判断だが、請求データ生成の上流としての整合は必須）。
- **適用性判断の根拠**: 請求命令によりオンライン請求（または限定的例外）が原則。薬局の電子レセプト請求猶予措置は平成23年3月31日に終了済みで、薬局は医科より早く電子請求が定着している。
- **UIへの含意（記録ライフサイクル）**:
  - 調剤記録→レセプト摘要への**データ一気通貫**（転記レス）が業務価値の中心。調剤録の記載事項とレセプト算定項目の整合バリデーションを入力時に行う。
  - 月次請求サイクル（毎月10日前後の請求締め）に合わせた「未請求/返戻/再請求」のステータスライフサイクル UI。返戻・査定結果の取込み表示は審査支払機関の運用に依存。
  - レセプト確定後の調剤録訂正は請求との不整合を生むため、確定後訂正には再請求フローへの誘導と警告が必要。

## 9. 調剤報酬上の医療 DX 体制評価（2026年度改定）

- **正式名称**: 医療DX推進体制整備加算（〜2026年5月）→ **電子的調剤情報連携体制整備加算**（2026年度調剤報酬改定で再編。名称は二次資料での確認であり、告示原文での最終確認を推奨）
- **発行主体**: 厚生労働省保険局（中医協答申・告示）
- **バージョン/改定日（確認できた内容）**: 医療DX推進体制整備加算の見直し（令和7年10月以降）: マイナ保険証利用率基準を 2025-10〜2026-02 は 60%、2026-03〜05 は 70% に段階引上げ。2026年度改定で「体制整備」から「利用実績」評価へ再編、**電子処方箋の応需体制＋紙処方箋分を含む調剤結果の原則全件・速やかな登録**が施設基準化。
- **確認日**: 2026-07-11
- **公式出典URL**:
  - https://www.mhlw.go.jp/content/12404000/001439601.pdf （厚労省: 医療DX推進体制整備加算の見直し（令和7年10月以降））
  - https://www.mhlw.go.jp/content/10200000/001257261.pdf （厚労省保険局医療課: 算定要件資料）
- **対象範囲**: 加算を算定する保険薬局（実質的にほぼ全数が対象化する見込み）。
- **適用性**: **Applicable**（算定は薬局の選択だが、市場要件として必須級）。
- **適用性判断の根拠**: 加算は任意算定だが、施設基準が電子処方箋・オン資・電子薬歴の運用実態を規定し、PH-OS のワークフロー設計に直結する。
- **UIへの含意**:
  - 「調剤結果登録率」「マイナ保険証利用率」等の**施設基準 KPI を管理画面でモニタリング**できると加算維持の実務価値が高い（利用率は審査支払機関からの通知値との照合が前提）。
  - 紙処方箋受付でも調剤結果登録を必須ステップとするワークフロー（§1 と同じ）。

## 10. 記録ライフサイクル設計への横断的含意（PH-OS 向けまとめ）

1. **保持期限マトリクスの一元管理**: 記録種別 × 起算日（調剤済み日/最終記入日/完結日）× 年限（3年 → 改正後5年、公費関連薬歴5年）を設定駆動にする。令和7年法律第37号の5年延長は**公布済み・施行日未確認**のため、ハードコード禁止・施行日追跡タスク化（Requires legal review）。
2. **追記型・監査証跡・確定操作**: e-文書法3要件（§6）が全法定記録の共通契約。PH-OS の Audit by Default 方針と整合するが、UI では「確定（finalize）」「訂正版追加＋理由」「作成責任者表示」「即時印刷ビュー」を記録系画面の標準部品にする。
3. **受付の二系統化と登録完了の可視化**: 電子/紙処方箋の受付、調剤結果登録の未完了ワークリスト、リフィル残回数管理が受付〜調剤ワークフロー UI の必須要素。
4. **在宅特化の差別化点**: 居宅同意取得型オンライン資格確認のモバイルフロー（訪問先での同意取得・資格確認・記録保存）は在宅訪問薬局向け SaaS として優先度の高い UI テーマ。
5. **削除・廃棄の統制**: 保持期限内の削除ブロック、期限後廃棄の監査ログ、患者からの開示請求対応（個情法）を記録詳細画面の共通アクション設計に含める。
