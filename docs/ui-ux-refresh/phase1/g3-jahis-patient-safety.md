### 2.1 JAHIS医療情報システムの患者安全ガイド〈内服外用編〉Ver.1.0

- 正式名称: JAHIS 医療情報システムの患者安全ガイド〈内服外用編〉Ver.1.0（JAHIS技術文書 14-102）
- 発行主体: 一般社団法人 保健医療福祉情報システム工業会 電子カルテ委員会 患者安全ガイドライン個別編WG
- バージョン: Ver.1.0（一覧上これが確認できた最新版。より新しい版は 2026-07-11 時点の制定済標準一覧に見当たらず）
- 公表日: 2014年11月
- 確認日: 2026-07-11
- 公式出典URL: https://www.jahis.jp/standard/detail/id=193 （全文PDF: https://www.jahis.jp/files/user/04_JAHIS%20standard/14-102_JAHIS%E5%8C%BB%E7%99%82%E6%83%85%E5%A0%B1%E3%82%B7%E3%82%B9%E3%83%86%E3%83%A0%E3%81%AE%E6%82%A3%E8%80%85%E5%AE%89%E5%85%A8%E3%82%AC%E3%82%A4%E3%83%89%EF%BC%9C%E5%86%85%E6%9C%8D%E5%A4%96%E7%94%A8%E7%B7%A8%EF%BC%9EVer.1.0.pdf ）
- 対象範囲: 内服薬・外用薬の処方業務を11段階（患者状態把握→処方入力→処方監査→疑義照会→処方指示受け→調剤→薬剤払出し→服薬指導・説明→薬剤交付（与薬）→持参薬管理→稼働前作業（マスタ設定等））に分解し、日本医療機能評価機構のヒヤリ・ハット事例分析に基づき、業務ごとの「リスク事例」「システム機能要件（最低限実装すべきもの）」「運用上の留意事項」を提示。電子カルテ/オーダエントリを前提とするが、処方監査・疑義照会・調剤・服薬指導・薬剤交付は薬剤業務そのもの。
- 入手制限: なし（公開PDF。引用明示で自由使用可、告知事項に明記）
- 適用性: **Applicable**（本調査で最もUI関連度が高い文書）
- 判断根拠: PH-OS の処方受付〜調剤〜監査〜服薬指導〜交付のワークフローと業務分解がほぼ一対一で対応。病院電カル前提だが、機能要件（薬剤検索・単位確認・チェック・変更中止可視化・交付時確認）は薬局システムにそのまま転写可能な抽象原理。
- UIへの含意（本文から確認した具体要件、確認日 2026-07-11）:
  - **薬剤検索（§7.2.3）**: 3文字以上での検索機能。濁点・半濁点を区別せず検索すると意図しない類似名薬剤（実事例: マイスリー/マイスタン）を誤選択するため「利用者の入力した通りの文字で検索」が望ましい。薬剤選択時に効能・メッセージ等を表示。薬効別一覧からの検索・選択を提供し、劇薬等の要注意薬剤は他剤と分けて誤選択を防ぐ。
  - **単位・用量（§7.2.3）**: 用量決定時にその薬剤に設定された単位を全て確認でき、選択された単位が選択後の画面で明示的に確認できること（mg/錠 取り違えで1日30錠入力の実事例）。
  - **入力チェック（§7.2.3）**: 同一患者への重複オーダ警告、用量チェック、薬剤間チェック（相互作用・禁忌）。定期/臨時・入院/外来・診療科をまたいで服用期間内でチェック。
  - **要注意薬の名称表示（§7.2.4）**: 表示名称の先頭に「麻薬:」等の明記またはマーク付与で選択ミス防止（表示名と印字名の二重管理に留意）。
  - **アレルギー（§7.1.3）**: アレルギー情報が「ある/なし/未確認」のいずれかを参照できること（未確認状態の明示が要件）。内容の確認、追加・変更の通知が望ましい。
  - **変更・中止の可視化（§7.5.3）**: 変更・中止があった指示は一覧画面等で一目で認識できること。変更・中止時は指示受けステータスの取消機能、当日変更患者の一覧警告表示、実施者への通知。
  - **交付時の一画面確認（§7.9.3）**: 用法等を数値コードで表さない・文字サイズを大きく読みやすく。指示内容は一画面でわかりやすく表示し、オーダ後の変更内容も含めて一画面で確認できること（処方画面とコメント画面の2画面分裂が与薬事故の実事例）。
  - **患者取り違え（§6）**: 「患者の取り違えはシステムでは防ぎにくい」とし、バーコード認証・患者に名乗ってもらう等の運用併用を提示（§7.9.4）。UIは正しい患者が選択されている前提を常時可視化する責務。
  - **マスタ設定（§7.11）**: マスタ修正間違い・上書き・登録漏れが重大事故要因。マスタ管理UIにもダブルチェック・変更監査の設計が必要。

### 2.7 JAHIS電子版お薬手帳データフォーマット仕様書 Ver.2.6

- 発行主体: 一般社団法人 保健医療福祉情報システム工業会
- バージョン: Ver.2.6（確認できた最新版）

## 3. 「ヒューマンエラー対策」名義の文書について

「ヒューマンエラー対策」を表題に持つ独立の JAHIS 文書は、2026-07-11 時点で制定済標準一覧・標準類マップから確認できなかった。JAHIS におけるヒューマンエラー対策は、(a) リスクマネジメントガイド〈解説編〉Ver.2.0 のユーザビリティ章（IEC 62366 系の使用エラー低減）、(b) 患者安全ガイド個別編（注射/内服外用/輸血/病理）の「リスク事例＋システム機能要件＋運用上の留意事項」の3点セット、(c) 処方オーダ共通化仕様ガイドライン（3文字検索・要注意薬表示等）に分散して規定されている、と整理するのが正確。

## 4. PH-OS UI への含意まとめ（抽象原理レベル）

1. **薬剤検索は3文字以上・入力どおり検索**: 前方一致の過短入力と濁点/半濁点正規化が類似名誤選択の実事故要因。PH-OS の薬剤マスタ検索（SSK/HOT）に「3文字未満は候補を出さない or 警告」「正規化は候補拡大でなく候補明示（正規化ヒットは区別表示）」を適用。
2. **単位の明示的再確認**: 用量入力後、選択単位（mg/錠/g/mL/枚…）を確定画面で常時可視化。単位違い（mg⇔錠）は最悪30倍投与級の実事例。
3. **チェック3点セット＋アレルギー未確認状態**: 重複投与・用量上限・相互作用/禁忌チェックを服用期間ベースで実施し、アレルギーは「あり/なし/未確認」の3状態を明示（未確認を「なし」に見せない）。CDS の YJ prefix 一致方針（既存実装）と整合。
4. **要注意薬の区分表示**: 麻薬・毒薬・劇薬・ハイリスク薬は名称先頭マーク/バッジで恒常表示（SSK の麻薬区分を活用）。警告色3段階ルール（重大/注意/情報）に区分をマップ。
5. **変更・中止の即時可視化**: 処方変更・中止は一覧レベルで一目で判別でき、下流ステータス（調剤・監査・交付）の取消/差戻しと通知を伴う。PH-OS のワークフロー状態色（差戻し=赤）と接続。
6. **交付前の一画面完結確認**: 交付（与薬）直前に患者・薬剤・用法・変更履歴を単一画面で確認できること。情報が2画面に分裂した構成自体が事故要因として記録されている。数値コード用法（朝=1等）の生表示禁止。
7. **患者取り違えはUI+運用の二重防御**: システムだけでは防ぎ切れない前提で、全画面共通の患者ヘッダ（PatientHeader SSOT）常時表示、交付時の患者名乗り/バーコード等の運用手順をガイド文言として支援。
8. **マスタ変更の安全設計**: 薬剤マスタ・用法マスタの修正/上書きUIに変更差分プレビュー・監査ログ・二重確認を備える（稼働前作業のヒヤリ・ハット最多領域）。
9. **警告設計の記録**: どのリスクをアラートで、どれを運用（ダブルチェック）で受容するかを、リスクマネジメントガイドの枠組み（リスク分析→コントロール→残留リスク）で docs/decisions.md に記録する。
