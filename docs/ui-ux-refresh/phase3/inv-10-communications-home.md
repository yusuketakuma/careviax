## De facto パターン・逸脱・重複実装（Phase 5 監査の種）

### De facto パターン（このバケットで確立している型）
- **page.tsx = server shell（PageScaffold + metadata.title + WorkflowPageHeader + Suspense/Loading）、'use client' content コンポーネントに実体**という構成がほぼ全ルートで共通。URL 初期状態は `*-query-state.ts`（requests/conferences/external/notifications）で SSR 読取して props 注入する型が確立。
- **状態表現の標準セット**: TanStack Query + `isLoading && !data`（initial loading と refetch の区別）+ `ErrorState`(onRetry) + `EmptyState` + toast(sonner) + mutation `isPending` disable。external の StatCard「—」表示は false-empty fail-close パターンの実装例（`external-viewer-content.tsx:431-462`）。
- **段階的エラー粒度**: dashboard-cockpit は `SegmentError` / `SegmentStaleBanner`（stale-refetch を明示する唯一級の実装、`dashboard-cockpit.tsx:1593-1605`）、external/インボックスはセクション独立 error。conferences/requests は画面単位 ErrorState。
- **確認ダイアログ規律**: referral-form の破棄/重複 ConfirmDialog（+ stale-ACK 無効化 `referral-form.tsx:178-186`、`useUnsavedChangesGuard`）、offline-sync の `ConflictDiffDialog`（keep/discard 明示）、settings の AlertDialog。CLAUDE.md エラー防止方針に沿う。
- **touch target**: `min-h-[44px]` / `!min-h-11` / `sm:h-11 sm:min-h-[44px]` が横断的に付与（Button variant contract と併用）。

### 逸脱
1. **offline-sync が PageScaffold / metadata.title 不使用**: `select-mode/page.tsx:5-6` に「全 route page は PageScaffold + metadata.title を必須とする(SSOT 4.4/7.9/2.2)」と明記があるのに、`offline-sync/page.tsx` は content を裸で return（title もなし）。バケット内唯一の明確な SSOT 違反候補。
2. **loading.tsx / error.tsx の colocation が不揃い**: conferences=両方あり、notifications=両方あり、dashboard=loading のみ、settings=error のみ、communications=error のみ（親階層）、inbound/requests/external/referrals/offline-sync/select-*=なし（グループ共通に依存）。基準が読み取れない。
3. **requests-content の empty がインラインテキスト**（`requests-content.tsx:321`「返信待ちの依頼はありません。」）で `EmptyState` 部品不使用。inbound/conferences/notifications/select-site は EmptyState を使用しており不統一。
4. **inbound だけ URL query-state ファイルがない**: フィルタ（channel/priority/status/selectedId）が useState のみで URL 共有・deep link 不可。兄弟ルート（requests/conferences/external/notifications）は query-state 型を持つ。
5. **dashboard-cockpit の xl 専用 3 カラム grid**（`dashboard-cockpit.tsx:1607` `xl:grid-cols-[280px_...]`）: UI_AUDIT_MATRIX T5 が指摘する「固定3カラムの非レスポンシブ(xl専用)」パターンの現存例（T5 の対象一覧には dashboard は未記載）。
6. **select-mode のエラーが toast のみ**: 画面内に error/retry 表示がなく、mutation 失敗時のリカバリ導線が toast 依存。ErrorState 標準から外れる。
