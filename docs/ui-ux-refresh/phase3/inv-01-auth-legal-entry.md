## このバケットで気づいた De facto パターン・逸脱・重複実装（Phase 5 監査の種。ここでは修正しない）

- **重複実装 1: `evaluatePasswordStrength` + `PasswordStrength` 型が 3 ファイルに完全コピー**。`src/app/(auth)/first-login/page.tsx:48`、`src/app/(auth)/password/change/page.tsx`、`src/app/(auth)/password/reset/page.tsx` — 同一ロジック・同一トークンマップ。基準変更時に 3 箇所同時修正が必要。
- **重複実装 2: 6桁コード入力ウィジェット（digits state + handleDigitChange ペースト処理 + Backspace 処理）が 3 ファイルにコピー**。`(auth)/mfa/page.tsx`、`(auth)/mfa/setup/page.tsx`、`(auth)/password/reset/page.tsx`。レイアウトが微妙に非統一（mfa/reset: `flex justify-center gap-2` + `h-12 w-12` / mfa/setup: `grid grid-cols-6` + `w-full min-w-0`）。
- **De facto パターン: auth カードシェルの手組み反復**。`section aria-labelledby` + `rounded-2xl border` + ヘッダ帯 `bg-slate-50/80` + pill バッジという構造が first-login/lockout/mfa/mfa-setup/password-change/password-reset で反復コピーされる一方、**login だけ別形状**（`rounded-lg`、`max-w-[29rem]`、ヘッダ帯なし）。共通コンポーネント不在。
- **生色クラス `bg-slate-50/80`** が auth ヘッダ帯全てに直書き（トークン化されていない唯一の背景色。ダークテーマ時の挙動未確認）。
- **middleware 保護のギャップ（要確認事項として記録）**: `PROTECTED_ROUTE_PREFIXES`（`src/proxy.ts:42`）のマッチは `=== prefix || startsWith(prefix + '/')` のため **`/dashboard-preview` は `/dashboard` に守られず edge 公開**。preview は「PHI-free で visibly non-production」が完成条件（UI_AUDIT_MATRIX family 行）だが、実体は本番 DashboardCockpit の再利用で、その公開性が意図か未確認。`/password/change`・`/mfa/setup` も同様に edge 公開（API 側認可依存 — 画面は誰でも開ける）。

- **Suspense fallback の粒度が画面ごとに別実装**: login は Skeleton + sr-only、shared は `Loading` label 付き。共通の「画面読み込み中」規約なし。
