## De facto パターン・逸脱・重複実装（Phase 5 監査の種）

### De facto パターン（このバケットで確立している流儀）
- **ワークスペース型 2 カラム**: `/reports` と `/billing` は「本文 + 右レール（次にやること/止まっている理由/根拠・記録）」構成で、`WorkspaceActionRail` / `GuardedWorkspaceActionRail` + 文言ルール（ブロッカー→「止まっている理由」）を共有。docstring で design SSOT（docs/design-gap-analysis-new.md の new_10/11）を明示（`report-share-workspace.tsx:45-50`, `billing-check-content.tsx:29-35`）。
- **fail-closed / false-empty 対策が新しめの画面の標準**: statistics（zod 負数 fail-closed・403 locked・「偽のラベル/数値は描かない」コメント）、report-delivery-dashboard（「取得失敗時は未確認報告がないものとして扱わず」）、GuardedWorkspaceActionRail（「問題なしではなく取得エラーです」）。MEMORY の FE False-Empty パターンが浸透。
- **印刷は必ず監査 POST → window.print**: `reports/[id]/print/page.tsx:534-590` と `print-hub-content.tsx:855,895` の両方が print audit（OCC: expected_report_updated_at）を通してからしか印刷しない。監査失敗＝印刷不可の fail-closed。
- **権限は API 返却 `permissions` オブジェクト駆動**: reports/[id]・share は can_edit/can_send/can_create_external_share/can_view_patient/can_create_followup_task をクライアントで参照。statistics のみサーバコンポーネントで hasPermission 直接判定という別流儀。

### 逸脱（要 Phase 5 判断）
1. **`/reports/analytics` が PageScaffold 不使用**（`reports/analytics/page.tsx:13` 生 `div p-3 md:p-4 xl:p-5`）。バケット内 10 ルート中これのみ。ヘッダも WorkflowPageHeader+WorkflowBackLink の独自組合せで、candidates/partner-cooperation の WorkflowPageIntro と機能重複。
2. **billing/candidates の confirm/exclude/reopen mutation に確認ダイアログなし**（ConfirmDialog/window.confirm 検出 0 件、`billing-candidates-content.tsx:483,722`）。partner-cooperation は同種の状態遷移に理由必須 ConfirmDialog を実装（`partner-cooperation-billing-content.tsx:1342-1397`）。CLAUDE.md「破壊的操作は確認ダイアログ必須」との整合が画面間で不統一。
3. **error 表現が 4 流儀併存**: ErrorState 共通部品（statistics/analytics/partner-coop/billing）、DataTable errorMessage+onRetry（candidates）、SegmentError（reports workspace）、**手書き border-l-4 state-confirm パネル**（`interprofessional-share-content.tsx:486-492,559-566` と `reports/[id]/page.tsx:769-773`）。後者 2 つは ErrorState 未使用の逸脱。
4. **StateBadge/StatusDot の適用が薄い**: 使用は share（×3）と candidates（×3）のみ。reports workspace・reports/[id]・print-hub は `bg-state-*` トークンを class 直書きしたローカル tone マップ（例 `report-share-workspace.tsx:60-61`、`print-hub-content.tsx:649-686`）。トークン自体は 6 軸 SSOT 準拠だがコンポーネント SSOT を通っていない。
5. **印刷帳票内の生 gray palette**: `reports/[id]/print/page.tsx:39-77` が `bg-gray-100 / bg-gray-800 / border-black` 等テーマトークン外の色を使用（紙出力前提の意図設計の可能性あり、未確認）。
6. **E2E 空白ゾーン**: `/reports/[id]/share`・`/reports/analytics`・`/reports/print`・`/statistics` は tools/tests 配下に該当 spec なし。unit は厚い（share 42 件・print-hub 58 件等）が実ブラウザ経路は未担保。
