# 患者同意書の文面・運用フロー

## 目的
在宅患者訪問薬剤管理指導および関連する多職種連携・外部共有を実施する前に、患者本人または適法な代理人から取得する同意の説明文面と、CareViaX 上の運用手順を固定する。

## 対象同意
- 訪問薬剤管理指導の実施同意
- 主治医、訪問看護、ケアマネジャー、施設、家族等への情報共有同意
- 電子的な記録保存、連携記録、監査証跡保持に関する説明

## 説明文面のたたき台
以下を患者向け説明書または同意書の本文に含める。

1. 利用目的
   - 在宅患者訪問薬剤管理指導、服薬支援、残薬確認、調剤後フォロー、疑義照会、多職種連携、報告書送付、請求根拠保存のために利用します。
2. 取得する情報
   - 氏名、生年月日、住所、連絡先、保険情報、処方内容、服薬状況、訪問記録、報告書、連携履歴、認証・監査ログを取り扱います。
3. 第三者提供
   - 法令上必要な場合、または患者本人の同意範囲内でのみ、主治医、訪問看護、ケアマネジャー、施設、家族等へ共有します。
4. 保存期間
   - 医療・薬歴・訪問関連記録および監査証跡は法令・ガイドラインに従い 5 年以上保存します。
5. 安全管理措置
   - 認証、権限管理、通信暗号化、監査ログ、バックアップ、アクセス制限を実施します。
6. 同意の撤回
   - 患者本人または代理人は、法令上継続保存が必要な範囲を除き、将来に向かって同意を撤回できます。

## CareViaX 運用フロー

### 1. 取得前準備
- 紹介受付または患者登録時に、必要書類として同意書の有無を確認する。
- 外部共有がある場合は、共有先の範囲を患者へ説明する。

### 2. 同意取得
- 対面、書面、または薬局規程で許容される方法で同意を取得する。
- 口頭同意のみで運用する場合でも、取得者、取得日時、説明対象、同席者を記録する。

### 3. CareViaX 登録
- 画面: `患者詳細 > 同意`
- API: `POST /api/consent-records`
- 必須項目:
  - `patient_id`
  - `consent_type`
  - `method`
  - `obtained_date`
- 推奨項目:
  - `expiry_date`
  - `document_url`
  - `case_id`

### 4. 訪問・請求前チェック
- `BillingEvidence` と workflow は有効同意を前提条件として判定する。
- 同意未取得または期限切れがある場合:
  - workflow の `consent_plan_huddle` に集約される
  - 請求候補は `missing_visit_consent` として block される

### 5. 期限管理
- 日次ジョブ `daily-consent-expiry` が 30 日前 / 7 日前の期限接近を通知する。
- API: `POST /api/jobs/daily-consent-expiry`
- 患者画面と workflow で再取得タスクを追う。

### 6. 撤回・更新
- 画面: `患者詳細 > 同意 > 撤回`
- API: `POST /api/consent-records/{id}/revoke`
- 撤回時は case 継続判断用の `WorkflowException` を起票し、共有・請求前提を再評価する。

## 運用メモ
- 外部共有トークンを発行する前に、共有範囲が同意内に含まれていることを確認する。
- 施設・家族向け共有は、共有先ごとの説明差分を残す。
- 期限がない同意でも、患者状態や共有範囲の変更時には再説明を行う。
