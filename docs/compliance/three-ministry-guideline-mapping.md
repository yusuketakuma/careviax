## 1. 目的と位置づけ

本文書は、PH-OS が実装するセキュリティ統制と、3省2ガイドライン（MHLW v6.0 + METI/MIC v1.1）の各統制項目との対応関係を明示する。外部監査・行政調査・ISMAP 審査において、PH-OS のコンプライアンス充足状況を説明するための一次エビデンスとして機能する。

### 充足状況の定義

| 充足状況     | 意味                                                     |
| ------------ | -------------------------------------------------------- |
| **充足**     | 統制要件を満たす実装が完了しており、運用中               |
| **部分充足** | 基本的な実装はあるが、設定・文書化・運用で改善余地がある |
| **計画中**   | 実装を計画しているが未完了                               |
| **対象外**   | PH-OS のアーキテクチャ・規模に照らして適用対象外         |

## 7. 監査対応ガイド

外部監査・行政調査の際に本文書を使用する際の手順を示す。

### 7.1 監査前の準備

1. 本文書（C-07）の最新版を確認する（最終更新日が6ヶ月以内であること）。
2. 「部分充足」項目の対応状況をアップデートする。
3. エビデンス一覧（Section 6）に記載の各ファイル・設定が最新状態であることを確認する。
4. AWS 設定のスクリーンショットまたは IaC（CDK/CloudFormation）の最新版を準備する。

### 7.2 監査員への説明ポイント

- **テナント分離**: PostgreSQL RLS による `org_id` ベースの分離は、アプリケーション層とDB層の二重防御（defense-in-depth）で実現している。
- **監査ログの不変性**: CloudTrail ログは S3 Object Lock（WORM）に保存され、アプリレベルのログは削除 API を持たない設計。
- **PHI の国内固定**: AWS SCP によりリージョン外へのデータ転送を技術的に禁止しており、設定ミスによる国外転送を防止している。
- **ISMAP 活用**: AWS ISMAP 認定により、インフラ層のセキュリティ要件（物理・論理・組織）は第三者監査済みであることを示す。
