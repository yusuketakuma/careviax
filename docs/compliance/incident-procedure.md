# インシデント手順書

## 目的
現場担当者がインシデント発生時に迷わず初動対応できるよう、実務手順を短く整理する。
詳細なエスカレーションと調査観点は `incident-response.md` を参照する。

## 初動チェックリスト
1. いつ、どの機能で、誰が、何をした時に発生したかを記録する。
2. 患者影響の有無を確認する。
3. 個人情報漏えいの疑いがあるか確認する。
4. 影響範囲を止めるため、該当操作・該当端末・該当アカウントを停止する。
5. インシデント責任者へ連絡し、チケットを起票する。

## 典型ケース別の対応

### 認証不能
- Cognito 障害か、個別アカウント問題かを切り分ける。
- MFA リカバリコード、管理者リセットの要否を確認する。
- 復旧までの間は代替手段で当日訪問情報を共有する。

### データ誤更新
- 変更対象の患者、ケース、訪問記録 ID を特定する。
- 監査ログで変更者・変更時刻を確認する。
- 影響データを保全し、必要に応じてリストアまたは再入力する。

### 文書送付失敗
- メール/FAX どちらで失敗したかを確認する。
- `DeliveryRecord` と `CommunicationEvent` を確認し、再送方針を決める。
- 緊急性が高い場合は電話等の代替チャネルで送達する。

### 端末紛失・盗難
- 該当端末の利用停止、パスワード変更、トークン失効を実施する。
- オフライン保存データの有無を確認し、影響評価を行う。

## 報告テンプレート
- 発生日時
- 発見者
- 事象概要
- 影響患者数 / 影響組織
- 暫定対応
- 追加支援の要否

## 収束条件
- 影響範囲が特定されている
- 暫定対応または恒久対応が完了している
- 必要な対外報告と社内報告が完了している
- 再発防止策の担当と期限が定義されている
