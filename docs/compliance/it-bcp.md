# IT-BCP

## 目的
災害、障害、サイバー攻撃、クラウド障害が発生した際に、PH-OS の重要業務を継続し、許容時間内に復旧するための方針を定める。

## 対象業務
- 当日訪問予定の確認
- 訪問前準備、持参物チェック、患者サマリー参照
- 訪問記録入力
- 報告書送付、連携履歴確認
- 算定根拠の保持と後続請求準備

## 復旧目標

| 項目 | 目標 |
|---|---|
| RTO（目標復旧時間） | 4 時間以内 |
| RPO（目標復旧時点） | 24 時間以内 |
| 優先復旧対象 | 認証、患者参照、訪問予定、訪問記録、報告書 |

## 想定シナリオ
- AWS 障害またはリージョン障害
- RDS 障害、データ破損、誤更新
- S3 障害、ファイル欠損
- Cognito 障害、認証不能
- 開発不具合による主要 API 停止
- 端末紛失、盗難、マルウェア感染

## 継続運用方針
1. 一時的にオンライン参照できない場合は、事前同期済みの訪問予定・患者要約を参照して訪問業務を継続する。
2. 新規処方取込、請求送信、外部システム同期は停止してよいが、訪問実施と最低限の記録は継続する。
3. 重要な連絡は電話・FAX・紙帳票の代替手段を利用する。
4. 復旧後は PH-OS を正本として再入力・再送・差分照合を行う。

## バックアップ方針
- RDS: 定期スナップショットと復旧手順を整備する。
- S3: バージョニングと保持設定を有効化し、重要文書は長期保存方針に従う。
- 監査ログ: 長期保管用アーカイブへ退避する。

## 復旧手順
1. 障害宣言と影響範囲の特定
2. 認証・DB・ファイル・通知の順に復旧優先度を判定
3. 代替運用中に発生した紙・口頭・オフライン記録を回収
4. 復旧後に差分入力、再送、請求候補再計算を実施
5. 収束判定後に RCA と改善策を記録

## 連絡体制
- インシデント責任者
- システム管理者
- 薬局管理者
- 外部ベンダー窓口

## 訓練
- 年 1 回以上の机上訓練
- 年 1 回以上の復旧試験
- 重要変更時の手順見直し
