## 2. CYCLE B — Pagination (keyset + merge floor)

### 2.1 Final cursor schema

New file `src/server/services/patient-detail-timeline-cursor.ts`. Opaque base64url JSON, versioned (`v:1`), Zod-decoded. **Malformed/stale → decode returns `null` (first page), never throws** — matching the `validationError` discipline at `route.ts:11`.

```ts
interface MergeFloor {
  // (occurred_at,id) of the LAST event emitted on the prior page.
  occurredAtMs: number; // last.occurred_at.getTime()
  id: string; // last prefixed event id, e.g. "inquiry:<uuid>"
} // USED ONLY to drop already-emitted rows at merge time — NEVER a DB WHERE bound.

type KeysetCol = { v: string | null; t: 'date' | 'time' | 'str' };

interface SourcePosition {
  cols: KeysetCol[]; // length === source.orderBy.length, same column order
  done?: boolean; // source returned < take (exhausted) OR was cap-frozen (§2.7)
}

export interface TimelineCursor {
  v: 1;
  floor: MergeFloor;
  pos: Partial<Record<string, SourcePosition>>; // keyed by SourceAdapter.key; ALL sources + 'operation_history'
```

### 2.2 Group-B mechanism — per-source keyset on the native orderBy tuple

The decisive fact (confirmed against `MAP:groupb`): a `findMany` with a seeked/larger window returns a **prefix/continuation of the same total `orderBy` order**, so seeking past source S's last-returned `orderBy` tuple is loss-free and dup-free **for S's own walk**, regardless of how S's `occurred_at` projects. The `occurred_at`/`orderBy` divergence (5 of 6 sources) affects only _merge placement_, which the global floor handles.

**Append `{ id: 'desc' }` to all 6 Group-B `orderBy` tuples** (op_history already has it at `patient-detail.ts:520`; precedent `externalShares` at registry:441). Verified necessary: `managementPlan` (registry:712), `firstVisitDocument` (registry:774), `prescriptionIntake` (registry:571) are single-column today → ties are non-deterministic → keyset would dup/skip. A total order is mandatory for keyset soundness.

**Two mandatory pins (flag in review):**

### 2.4 Seek / window / next_cursor (orchestrator in `patient-detail.ts`, replacing `:549-559`)

```ts
const PAGE_SIZE = 40;
const cursor = decodeCursor(args.cursor); // {floor,pos} or null (first page / malformed)

const fetched = await Promise.all(
  TIMELINE_SOURCES.map(async (s) => {
    const prev = cursor?.pos[s.key];
    if (prev?.done) return { key: s.key, result: { rows: [], done: true }, events: [] };
    const result = await s.fetch({ ...fetchCtx, position: prev });
    return { key: s.key, result, events: s.toEvents(result.rows, projectCtx) };
  }),
);

const allEvents = [
  ...fetched.flatMap((f) => f.events),
  ...buildOperationHistoryEvents(operationHistory, projectCtx),
]; // §2.5

// (a) drop rows at/above prior floor (removes over-fetched rows already emitted)
const floor = cursor?.floor;
const floored = allEvents.filter(
  (e) =>
    !floor ||
    e.occurred_at.getTime() < floor.occurredAtMs ||
    (e.occurred_at.getTime() === floor.occurredAtMs && e.id.localeCompare(floor.id) < 0),
);

// (b) same global sort as today, then page
const sorted = floored.sort(
  (l, r) => r.occurred_at.getTime() - l.occurred_at.getTime() || r.id.localeCompare(l.id),
);
const page = sorted.slice(0, PAGE_SIZE);

// (c) new floor = last emitted event
const last = page.at(-1);
const nextFloor = last
  ? { occurredAtMs: last.occurred_at.getTime(), id: last.id }
  : (floor ?? { occurredAtMs: 0, id: '' });

// (d) advance EACH source to its LAST DB row (not the merge cut). Over-fetched remainder is safe:
//     next page's (a) floor-drop removes any whose occurred_at sits above the new floor.
const nextPos: TimelineCursor['pos'] = {};
for (const f of fetched) {
  const prev = cursor?.pos[f.key];
  if (f.result.done && f.result.rows.length === 0) {
    nextPos[f.key] = { ...prev, done: true };
    continue;
  }
  nextPos[f.key] = { cols: f.result.nextPosition?.cols ?? prev?.cols ?? [], done: f.result.done };
}

// (e) cursor null only when page is short AND every source exhausted
const next_cursor =
  page.length < PAGE_SIZE && TIMELINE_SOURCES.every((s) => nextPos[s.key]?.done)
    ? null
    : encodeCursor({ v: 1, floor: nextFloor, pos: nextPos });

return {
  timeline_events: page,
  self_reports: selfReports,
  next_cursor,
  ...(partialFailures.length > 0 ? { partial_failures: partialFailures } : {}),
};
```

**Why (d) is correct (load-bearing):** a source's `take` rows may overflow the page's top-40. The over-fetched remainder has `occurred_at` values that, on the next page, are **above** the new floor and dropped by (a). So those rows are never lost (already shown OR floor-dropped), never re-pulled (keyset seeks past the DB tail), never re-emitted (floor). This is the entire reason for the two-part cursor.

```ts
export type ScopedTxRunner = <T>(work: (tx: Prisma.TransactionClient) => Promise<T>) => Promise<T>;

export function createScopedTxRunner(
  orgId: string,
  options?: {
    requestContext?: RequestAuthContext;
    client?: { $transaction: PrismaTx['$transaction'] };
    timeoutMs?: number;
  },

  validateOrgId(orgId); // reuse :34
  return (work) => {
    const requestContext = options?.requestContext ?? getRequestAuthContext(); // reuse :65
    if (requestContext?.orgId && requestContext.orgId !== orgId)
      // reuse :66-68

        `Request orgId mismatch: expected ${requestContext.orgId}, received ${orgId}`,

    if (!requestContext) logSecurityEvent({ event_type: 'rls_context_missing' /* …:70-78 */ });
```
